The Leagues feature introduces a weekly competitive system where players earn points through matches to climb through six levels. Performance determines whether a player is promoted, stays in their current league, or is relegated, with the leaderboard resetting every Monday.

Players compete within their own league as well as on a global scale. The League Leaderboard tracks performance within a specific tier, while the Global Leaderboard highlights the highest-scoring players across all leagues, showcasing the best of the best.

The leagues implement a sophisticated UI focused mainly on leaderboard components while maintaining a lightweight graphical footprint. Instead of instantiating individual game objects for every leaderboard entry, which would be resource-intensive, I adopted a data-driven approach. By leveraging the EnhancedScroller plugin, I efficiently recycle scroll rect elements, enabling the leaderboard to dynamically render an effectively unlimited number of entries with minimal performance overhead.

At the end of each week, players receive rewards based on their final ranking in their league. Higher positions offer better rewards, which are distributed throughout the following week, incentivizing consistent and competitive play.

We wanted to create visually engaging animations to present the outcome of the competition to users. The animation needed to clearly convey whether the player was promoted, demoted, or remained in the same league. I aimed to incorporate 3D elements of the league shields to as a personal experiment. However, integrating 3D visuals into this project wasnâ€™t straightforward. Since the entire application is UI-driven, the main camera is configured without a 3D perspective, and introducing one would require significant architectural changes.

To enable 3D rendering of the shields, I first modeled each one in Blender. After importing them into Unity, I applied a simple unlit material to the meshes. To display them within the UI, I set up a separate off-screen camera that renders the 3D models to a RenderTexture. This texture is then assigned to a RawImage component in the UI. This setup allows the 3D shields to be animated in the scene while remaining visually integrated with the existing UI system.

To complete the visual presentation, I created custom particle systems to add impactful finishing effects.
